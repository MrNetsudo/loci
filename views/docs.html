NetSudo Documentation

Complete internal reference for the NetSudo agency platform. Covers the CRM, client portal, Loci chat app, billing, security, and API.

Overview

NetSudo is a full-stack agency management platform built for digital agencies. It provides a CRM with pipeline management, a client-facing portal, invoicing, proposals, booking, and real-time Telegram notifications — all running on a single Express.js server backed by PostgreSQL.

Loci is a venue-based real-time chat application built on Supabase. Users discover nearby venues and join chat rooms. Loci runs alongside NetSudo on the same infrastructure.

Full REST API for contacts, revenue, projects, users, and settings with session auth.

Quick Start

Getting up and running with NetSudo for development or staff access.

Logging In

  1. Navigate to the admin panel
    Go to https://netsudo.com/admin/login in your browser.
  2. Enter your credentials
    Use the username and password provided by your administrator. Passwords are case-sensitive.
  3. Complete 2FA (if enabled)
    If your account has two-factor authentication enabled, enter the 6-digit code from your authenticator app (Google Authenticator, Authy, etc.).
  4. Access the dashboard
    After login, you'll be redirected to the dashboard showing total contacts, pipeline summary, and recent activity.
Account Lockout After 5 consecutive failed login attempts, your account is locked for 15 minutes. A Telegram notification is sent to the administrator. Contact your admin if locked out.

Client Portal

The client portal gives your clients a branded, secure space to interact with their projects without needing admin access.

Features

  • Project Overview — Clients see their project status, milestones, and deliverables in real time
  • Messaging — Threaded conversations between client and team, stored in the activity log
  • File Sharing — Upload and download project assets (designs, documents, code deliverables)
  • Invoice Access — View and pay invoices directly through the portal via Stripe
  • Proposal Review — Review proposals with scope, timeline, and pricing — accept or request changes

How Clients Get Access

  1. Admin creates the client contact
    Add the client in the CRM with their email address. Move them to an active stage.
  2. Portal invite is sent
    Send a portal invitation via email (Resend). The client receives a unique login link.
  3. Client sets their password
    On first login, clients set their own password. No admin password sharing needed.
  4. Ongoing access
    Clients log in at the portal URL and see only their own projects, invoices, and messages.
Data Isolation Clients can only see data associated with their own account. All queries are scoped by contact_id. There is no cross-client data leakage.

Loci Platform

Loci is a venue-based real-time chat application. Users can discover nearby venues (bars, cafes, events) and join chat rooms attached to those locations.

How It Works

  1. User opens Loci
    Available at loci.netsudo.com. Users can sign up with email (OTP verification) or use anonymous access.
  2. Location detection
    The app uses browser geolocation to find nearby venues. Venues are displayed on a map or list sorted by distance.
  3. Join a room
    Each venue has chat rooms. Users join a room and see real-time messages from other users at that venue.
  4. Real-time messaging
    Messages are delivered instantly via Supabase Realtime (WebSocket channels). No polling.

Billing & Invoicing

NetSudo includes a full billing system for tracking revenue, managing projects, and sending invoices to clients.

Revenue Tracking

Revenue entries are tracked in the CEO dashboard with two types:

TypeDescriptionMRR Impact
one-timeProject payments, setup fees, one-off workNot included in MRR
recurringMonthly retainers, hosting, maintenanceCounted toward MRR goal

Revenue Statuses

  • pending — Invoice sent but not yet paid
  • paid — Payment received and confirmed

Project Management

Projects are linked to contacts and track:

  • Value — Total project value (one-time)
  • Monthly Value — Recurring monthly revenue from this project
  • Status — Active, completed, paused, or cancelled
  • Dates — Start date and deadline tracking
  • Notes — Internal project notes

Invoicing Workflow

  1. Create invoice
    Build an invoice with line items, quantities, and prices. Link it to a contact and project.
  2. Send to client
    Email the invoice via Resend. The client receives a branded email with a link to view and pay.
  3. Client pays
    Client views the public invoice page and pays via Stripe. Payment is processed securely.
  4. Revenue recorded
    Once paid, the revenue entry is marked as paid and reflected in the CEO dashboard and weekly reports.

CEO Dashboard

The CEO dashboard (/admin/ceo) provides executive-level metrics:

  • Month Revenue — Total paid revenue for the current month
  • MRR — Monthly recurring revenue from all recurring entries
  • Goal Progress — Visual progress toward the $50K MRR goal
  • Pipeline Value — Total value of active deals in the pipeline
  • Daily Revenue Chart — Day-by-day revenue for the current month
  • Active Clients — List of clients with recent activity

Weekly Telegram Reports

Every Monday at 9 AM EST, NetSudo automatically sends a weekly report via Telegram including:

  • Revenue this month
  • MRR vs goal with percentage
  • Gap to goal (how much more is needed)
  • Pipeline deals count and total value
  • New contacts this week
