API Reference

Complete endpoint reference for the NetSudo Admin API, Client Portal API, and Loci API. All examples use curl.

Authentication The NetSudo API uses session-based authentication. Log in via POST /admin/login to receive a connect.sid session cookie. Include this cookie in all subsequent requests. There are no API keys or bearer tokens.
Base URLhttps://netsudo.com
AuthSession cookie (connect.sid)
Content-Typeapplication/json (except login: application/x-www-form-urlencoded)
Rate LimitLogin: 10 requests / 15 min per IP

Admin API

Core API for the NetSudo admin panel. All /admin/api/* endpoints require session auth unless noted.

Authentication

MethodPathDescriptionAuth
POST/admin/loginAuthenticate with username/passwordPublic (rate limited)
POST/admin/2faSubmit TOTP code after loginPending session
GET/admin/logoutDestroy session and redirect to loginSession
GET/admin/api/meGet current authenticated userSession

Login Example

bash 
# Login and save session cookie
curl -X POST https://netsudo.com/admin/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=yourpassword" \
  -c cookies.txt \
  -L

# If 2FA is enabled, submit the TOTP code
curl -X POST https://netsudo.com/admin/2fa \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "token=123456" \
  -b cookies.txt \
  -c cookies.txt \
  -L

# Check current session
curl https://netsudo.com/admin/api/me \
  -b cookies.txt

List Contacts

bash 
# List all contacts
curl https://netsudo.com/admin/api/contacts \
  -b cookies.txt

# Filter by stage and search
curl "https://netsudo.com/admin/api/contacts?stage=New&search=acme" \
  -b cookies.txt

# Paginate results
curl "https://netsudo.com/admin/api/contacts?limit=20&offset=40" \
  -b cookies.txt

Create Contact

bash 
# Create a new contact (authenticated)
curl -X POST https://netsudo.com/admin/api/contacts \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "name": "Jane Smith",
    "email": "jane@example.com",
    "company": "Acme Corp",
    "phone": "+1-555-0100",
    "stage": "New",
    "tag": "lead",
    "notes": "Referred by existing client"
  }'

# Website form submission (public, no auth)
curl -X POST https://netsudo.com/admin/api/contacts \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Web Visitor",
    "email": "visitor@example.com",
    "message": "Interested in your services",
    "source": "website"
  }'

Update Contact & Stage

bash 
# Update contact details
curl -X PUT https://netsudo.com/admin/api/contacts/42 \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "name": "Jane Smith",
    "email": "jane@acme.com",
    "company": "Acme Corp",
    "phone": "+1-555-0100",
    "stage": "In Conversation",
    "tag": "prospect",
    "notes": "Had great intro call"
  }'

# Quick stage update (used by Kanban drag-and-drop)
curl -X PATCH https://netsudo.com/admin/api/contacts/42/stage \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"stage": "Closed Won"}'

# Add a note
curl -X POST https://netsudo.com/admin/api/contacts/42/note \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"note": "Follow up scheduled for next Tuesday"}'

# Export all contacts as CSV
curl https://netsudo.com/admin/api/contacts/export/csv \
  -b cookies.txt \
  -o contacts.csv

Users

Superadmin Only All user management endpoints require the superadmin role. Regular admins will receive a 403 Forbidden response.
MethodPathDescriptionAuth
GET/admin/api/usersList all admin usersSuperadmin
POST/admin/api/usersCreate a new admin userSuperadmin
PUT/admin/api/users/:idUpdate user (username, role, password)Superadmin
DELETE/admin/api/users/:idDelete an admin userSuperadmin

User Management Examples

bash 
# List all admin users
curl https://netsudo.com/admin/api/users \
  -b cookies.txt

# Create a new admin user
curl -X POST https://netsudo.com/admin/api/users \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "username": "sarah",
    "password": "SecurePass123!",
    "role": "admin"
  }'

# Update user role to superadmin
curl -X PUT https://netsudo.com/admin/api/users/5 \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "username": "sarah",
    "role": "superadmin"
  }'

# Delete a user (cannot delete yourself)
curl -X DELETE https://netsudo.com/admin/api/users/5 \
  -b cookies.txt

Settings & 2FA

MethodPathDescriptionAuth
POST/admin/api/settings/passwordChange current user's passwordSession
GET/admin/settings/2faGet 2FA status (+ QR code if not enabled)Session
POST/admin/settings/2fa/enableEnable 2FA with TOTP verificationSession
POST/admin/settings/2fa/disableDisable 2FA (requires password)Session

Settings Examples

bash 
# Change password
curl -X POST https://netsudo.com/admin/api/settings/password \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "currentPassword": "OldPass123",
    "newPassword": "NewSecurePass456!"
  }'

# Check 2FA status (returns QR code if not yet enabled)
curl https://netsudo.com/admin/settings/2fa \
  -b cookies.txt

# Enable 2FA after scanning QR code
curl -X POST https://netsudo.com/admin/settings/2fa/enable \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"token": "123456"}'

# Disable 2FA (requires current password)
curl -X POST https://netsudo.com/admin/settings/2fa/disable \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"password": "YourCurrentPassword"}'

Loci API

The Loci API handles venue-based real-time chat. It uses Supabase for auth, realtime channels, and data storage. The beta signup endpoint integrates with the NetSudo CRM.

Loci Authentication

MethodPathDescriptionAuth
POST/auth/signupRegister with email (sends OTP)Public
POST/auth/verifyVerify email OTPPublic
POST/auth/anonymousRegister anonymouslyPublic

Loci Data Endpoints

MethodPathDescriptionAuth
GET/api/venues/nearbyFind venues near coordinatesSupabase token
GET/api/rooms/:venue_idGet rooms for a venueSupabase token
GET/api/messages/:room_idGet messages in a roomSupabase token
POST/api/messagesSend a message to a roomSupabase token
WSrealtime/room:{room_id}Subscribe to room messagesSupabase token

NetSudo Integration

MethodPathDescriptionAuth
POST/admin/loci/signupLoci beta signup (creates CRM contact)Public
POST/admin/loci/qrUpdate Loci QR code URLSecret key
GET/admin/loci/qrGet current Loci QR code URLPublic

Loci Integration Examples

bash 
# Submit a Loci beta signup (goes into NetSudo CRM)
curl -X POST https://netsudo.com/admin/loci/signup \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Alex Johnson",
    "email": "alex@example.com"
  }'

# Update Loci QR URL (requires secret)
curl -X POST https://netsudo.com/admin/loci/qr \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://loci.netsudo.com/download",
    "secret": "LociQR2026!"
  }'

# Get current QR URL
curl https://netsudo.com/admin/loci/qr
